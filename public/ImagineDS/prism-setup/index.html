Team Onboarding

Set up the Prism Design System

Download the two config files, place them in the right folder, and Claude will know about the design system in every project you open.

Step 1 — Download the files

palette

prism-design-system.md

The full design system — all tokens, typography, spacing, colors, and rules. Claude reads this in every project.

~/.claude/prism-design-system.md
settings

CLAUDE.md

The global Claude config file. Imports the design system and has a space for your personal preferences.

~/.claude/CLAUDE.md

Step 2 — Manual setup (one time only)

1

Create the ~/.claude/ folder if it doesn't exist

Open Terminal and run:

mkdir -p ~/.claude
2

Place prism-design-system.md in ~/.claude/

Move the downloaded file to the correct location:

mv ~/Downloads/prism-design-system.md ~/.claude/prism-design-system.md
If you downloaded it somewhere else, replace ~/Downloads/ with the actual path.
3

Set up CLAUDE.md

If you don't have a ~/.claude/CLAUDE.md yet: move the downloaded file there:

mv ~/Downloads/CLAUDE.md ~/.claude/CLAUDE.md

If you already have a ~/.claude/CLAUDE.md: open it and add this as the very first line:

@~/.claude/prism-design-system.md
Do not overwrite your existing CLAUDE.md — just add the import line at the top.
4

Install PP Neue Montreal fonts system-wide

Copy these 6 .otf files from your font folder to ~/Library/Fonts/:

  • PPNeueMontreal-Light.otf
  • PPNeueMontreal-Regular.otf
  • PPNeueMontreal-Book.otf
  • PPNeueMontreal-Medium.otf
  • PPNeueMontreal-SemiBold.otf
  • PPNeueMontreal-Bold.otf

Or in Terminal (replace the path with where your fonts are):

cp /path/to/your/fonts/PPNeueMontreal-*.otf ~/Library/Fonts/
These are the licensed PP Neue Montreal fonts you already have. If you don't have them, ask the team admin.
5

Copy logo SVGs to ~/.claude/logos/

Create the logos folder and copy the SVG files there:

mkdir -p ~/.claude/logos cp /path/to/your/logos/*.svg ~/.claude/logos/
The logos folder gives Claude access to brand assets for any project. Replace the path with where your logo SVGs are stored.

Step 3 — Verify with Claude

smart_toy Open any project folder in Claude Code and run this prompt

Please confirm my Prism Design System setup is working: 1. Read ~/.claude/prism-design-system.md and confirm it's loaded 2. Tell me which font, color tokens, and spacing tokens are in the system 3. Confirm ~/.claude/CLAUDE.md exists and imports the design system

info Claude will confirm the design system is active. From this point, every project you open in Claude Code will automatically have access to all Prism tokens, typography, and rules.

Step 4 — For every imagine.io project

1

Create a CLAUDE.md in the project root

The Prism Design System only activates when Claude sees this file in the project. Without it, Claude will not apply any Prism rules — no font, no tokens, no logos.

Create a file called CLAUDE.md in the root of the project with exactly this content:

This is an imagine.io project — apply the Prism Design System.
You can create this file manually, or ask Claude at the start of any new project: "Create a CLAUDE.md for this imagine.io project" and it will do it for you.

Keeping in sync with Figma

sync When the design system updates in Figma

Whenever tokens, colors, or typography change in the Figma file, anyone on the team can re-sync their local file by running this exact prompt from any project in Claude Code. Claude will update ~/.claude/prism-design-system.md automatically — your personal preferences in CLAUDE.md will not be touched.

Re-sync ~/.claude/prism-design-system.md with the latest Prism Design System from this Figma file: https://www.figma.com/design/kiwttYrNajH3hQwrtGtrle/Prism-Design-System?node-id=1-3
