# Page Spec: /resources

## Page Objective
Serve as the top-level content hub for all educational, strategic, and editorial content from Imagine. Help visitors find the right content format for their current need — whether that is a quick blog post, a deep-dive guide, a customer case study, or a product update.

## Audience
- Ecommerce and digital leads at configurable product brands evaluating Imagine
- Technical leads and developers looking for educational content on configurable commerce
- Existing customers looking for product updates and new guides
- Analysts and category researchers building context on the configurable commerce space

## Key Message
Imagine publishes thinking that helps brands and developers build better configurable commerce experiences — whether or not they are using the platform. The resources hub is a signal of category authority, not a brochure.

## Copy Notes
- The resources hub should feel like a well-edited publication index, not a CMS dump.
- Featured content should be manually curated and updated regularly.
- Content type descriptions should explain the format and purpose, not just label it.
- Newsletter copy should be honest about frequency and content — no vague "latest news" language.

## Design Notes
- Filter bar should be sticky on scroll for easy content-type navigation
- Featured picks displayed as large editorial cards (not small thumbnails)
- Content type grid: each section visually distinct but consistent in structure
- Newsletter section: minimal, trust-forward — no imagery, just the value prop and form
- Tags (type, audience) should be consistent across all content cards
